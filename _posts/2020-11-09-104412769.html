Jira Software 8.13.x 릴리즈 노트

| Get the Jira Server app on managed devices / 관리되는 장치에서 Jira 서버 앱 가져오기

이제 MDM(Mobile Device Management) 솔루션을 사용하여 Jira Server 모바일 앱을 팀에 배포할 수 있습니다. 이 기능은 앱을 회사에서 승인한 장치 또는 특정 사용자로 제한해야할 경우에 유용합니다.


MDM 솔루션이 AppConfig 표준을 지원하는 경우 앱에 사이트 URL을 미리 입력하여 사용자의 시간을 절약하고 실수를 방지할 수 있습니다. 사이트가 많다면 여러 사이트 URL을 미리 등록하여 로그인이 더 간편해집니다. 자세히 알아보기

| Hungry for new features? / 새로운 기능을 원하시나요?

장기 지원 릴리즈였던 지라 8.5.x를 출시한 이후 많은 변화가 있었습니다. 이슈 아카이빙, 향상된 성능 및 접근성, 그리고 지라에서 더 나은 서비스를 제공하기 위해 많은것들을 변경하였습니다. 변경 로그 에서 JIRA 7.13에서 8.13 사이의 모든 변경 및 수정사항을 확인하실 수 있습니다.

| API change log / API 변경 로그

지난번 장기지원 (Long Term Support) 릴리즈 이후로 변경된 사항에 대한 전체 정보를 얻을 수 있도록 REST API 변경 로그를 준비했습니다. 자세히 알아보기

| Long Term Support releases, performance-wise / 장기 지원 릴리스, 성능 측면

모든 지라 릴리스에 대해 최신 버전을 이전 버전과 비교하기 위해 광범위한 성능 테스트를 실행합니다. 새로운 기능이 지라에게 어떤 영향을 미치는지 알아보기 위한 것이며, 성능의 저하가 있지는 않은지 확인하기 위한 것입니다.

Jira 장기 지원 릴리즈 간의 성능도 비교하고 있습니다. 많은 변경점이 있었기 때문에  개선 사항이 훨씬 더 눈에 띕니다.

성능 및 스케일 테스트 에서 Jira 8.5 LTS, Jira 8.13 LTS 간의 성능 비교 결과를 볼 수 있습니다.

| Always a good time to clean up Jira / 언제나 지라를 정리하기 좋은 시간

지라가 느려지고 있고, 사용자들이 원하는 것을 찾을 수 없다고 느낀다면 정리 프로젝트를 시작하고 팀 생산성을 높여보세요. 이에 대한 팁과 요령은 Jira 정리 가이드 를 확인해보세요.



Jira Service Desk 4.13.x 릴리즈 노트

| Hungry for new features? / 새로운 기능을 원하시나요?

장기 지원 릴리즈였던 지라 8.5.x, 지라 서비스 데스크 4.5 를 출시한 이후 많은 변화가 있었습니다. Opsgenie와 Conluence Cloud와의 통합을 소개하고, 에이전트 대기열을 개선하고, 새로운 플랫폼을 추가하고, 일부를 제거 하였습니다. 또한 지라에서 더 나은 서비스를 제공하기 위해 많은것들을 변경하였습니다. 마지막 장기 지원 (LTS) 릴리스 간에 발생한 모든 변경 및 수정 사항을 확인해보세요.

Confluence 7.8 릴리즈 노트

| Capture more end-user activity in the audit log / 감사 로그에서 최종 사용자의 더 많은 활동기록 캡처

[ Data Center ]

이제 누군가 검색을 수행할 때 감사 로그에 기록될 수 있습니다. 이 이벤트는 검색 필드, 고급 검색 및 라이브 검색, 페이지 트리 검색과 같은 검색 매크로에 입력한 검색어를 기록합니다.

'수행된 검색' 이벤트는 전체 적용 범위 수준에 포함되며, 데이터 센터 라이센스에서 제공됩니다.  검색어 데이터를 수집하지 않으려면 시스템 속성에서 이 이벤트를 비활성화할 수 있습니다.

| Increased label limit for Page Properties Report macro / 페이지 속성 리포트 매크로의 레이블 한계수치 증가

[ Server ] [ Data Center ]

페이지 속성 보고서 매크로는 레이블에 의존하여 여러 공간에 걸쳐 페이지를 구성합니다. 기존 매크로에는 레이블은 20개로 제한되어 있어 공간과 페이지의 설계가 제한되어 있었습니다.

매크로 브라우저에 최대 60개의 레이블을 입력할 수 있도록 제한을 늘렸습니다. 이제 모든 페이지를 Reporting하는것이 더 쉬워졌습니다.

| Disabled macros will preserve your content / 비활성화된 매크로가 콘텐츠를 보존합니다.

[ Server ] [ Data Center ]

매크로는 가장 강력한 Confluence의 기능 중 하나입니다. 이전에는 본문 내용이 있는 매크로를 사용하지 않도록 설정하면 내용이 손실되었었지만. 이제 비활성화된 매크로가 본문 내용을 유지하면서 편집기에 표시할 수 있도록 지원합니다.

뷰 페이지 모드에서 비활성화된 매크로는 여전히 placeholder image(알 수 없는 매크로)를 표시합니다. 뷰 모드로 콘텐츠를 표시하려면 편집기에서 비활성화된 매크로를 선택하고 내용을 표시할 수 있는 새 매크로로 변경하시면 됩니다. 본문 내용이 새 매크로에서 유지되고 보기 모드로 표시됩니다.

| Allowlist and blocklist replace whitelist and blacklist / 용어 변경

[ Server ] [ Data Center ]

단어의 선택은 우리의 다양한 팀들을 하나로 묶어주는 구성 요소들중 하나입니다. '블랙리스트' '화이트리스트' 등의 용어는 흑백은 반대, 하나는 나쁘고 다른 하나는 좋다는 개념을 강화하므로 이를 변경하였습니다.

"허용리스트"와 "차단리스트"는  "화이트리스트"와 "블랙리스트" 보다 명확하고 직접적인 용어입니다. 사용자 인터페이스에서 확인할 수 있습니다. API는 변경되지 않았으므로 화이트리스트 및 블랙리스트는 코드베이스에 남아 있습니다.



Crowd 4.2 릴리즈 노트


| Passwords to external systems now encrypted / 이제 외부 시스템의 비밀번호가 암호화됩니다.

보안을 강화하기 위해 지금까지 Crowd 데이터베이스에 일반 텍스트로 보관되어 온 외부 시스템의 모든 비밀번호는 Crowd 4.2부터 암호화됩니다. 업그레이드 예정인 사용자일 경우 업그레이드 중에 비밀번호 암호화가 수행됩니다.

다음은 암호화된 중요한 데이터 목록입니다.

  • LDAP 디렉토리 암호

  • 원격 Crowd 디렉터리 응용 프로그램 암호

  • Azure AD 웹 응용 프로그램 키

  • SMTP 메일 암호

  • 프록시 암호

Crowd의 비밀번호 암호화에 대한 자세한 내용은 비밀번호 암호화 를 참조하십시오.



| New attribute mapping when importing users via CSV / CSV를 통해 사용자를 가져올 때의 새 특성 매핑

불필요한 라이센스 사용을 방지하기 위해 Crowd 4.2를 사용하면 CSV 파일을 사용하여 사용자를 활성 또는 비활성 상태로 가져올 수 있습니다. CSV 파일을 사용하여 사용자를 가져올때 다음 세 가지 상태 중 하나를 선택할 수 있습니다.

  • 활성화된 (active)
  • 비활성화된 (inactive)
  • CSV 파일과 동일하게




| Look and feel / 외관과 느낌

Crowd 4.2버전 부터 로그인 화면의 모양과 느낌을 커스터마이징할 수 있습니다. 사용자의 로고, 사용자 정의 시작 텍스트 및 선택한 색상을 변경하여 페이지를 커스터마이징할 수 있습니다. 이렇게 하면 Crowd 인스턴스를 돋보이게 할 수 있고, 회사 색상으로 Crowd 인스턴스를 변경하거나, 로그인 화면을 변경하는 방법에 대한 자세한 내용은 외관 및 느낌 을 확인해보세요.




| Improvements to directory details and audit log screens / 디렉터리 세부 정보 및 감사 로그 화면 개선

Crowd 4.2는 최신 동기화의 상태를 빠르게 확인할 수 있도록 디렉토리 세부사항과 감사 로그 화면을 새로 고쳤습니다. 동기화 오류가 발생할 경우 원인을 더 잘 파악할 수 있는 방향으로 변경되었습니다. 변경사항은 다음과 같습니다.

  • 디렉터리 보기에서 다음을 추가했습니다.
    • 마지막 동기화 상태에 대한 확장된 정보
    • 노드 ID 및 노드 이름(Crowd Data Center에서만 사용 가능).
  • 감사 로그에 다음을 추가했습니다.
    • 증분 및 전체 동기화 정보,
    • 노드 정보(Crowd DC에서만 사용 가능),
    • 동기화에 걸리는 시간에 대한 정보
    • 성공/실패 빌드에 대한 그래픽 지표를 지웁니다.




| Performance improvements / 성능 향상

Crowd 4.2는 Crowd에서 어플리케이션으로의 전체 동기화의 성능을 향상 시켰습니다. 멤버십을 가져오는 속도가 지금은 10배 정도 빨라져 전체 동기화 시간이 평균 절반으로 줄어들었습니다.



Bitbucket Server 7.7 릴리즈 노트

이번 Bitbucket Server 및 Data Center 릴리스에서는 워크플로우를 개선하였습니다.  비활성화된 pull request 는 설정한 기간(주)이 지난뒤 자동으로 거부됩니다. Pull request를 리뷰할 때 diff 뷰는 더 보기 쉬운 색상옵션으로 변경할 수 있게 되었고 코멘트에 이모지반응을 추가할 수 있게 되었습니다.


| New code review workflow - Batch and publish comments / 새 코드 리뷰 워크플로우 - 배치 및 주석 게시

[ Server ] [ Data Center ]

Pull request 리뷰어로서 검토 프로세스 중에 파일과 코드에 대한 여러 코멘트들을 작성할 수 있습니다 . 준비가 되면 한 번에 하나씩 코멘트를 보내는 것이 아니라 한 번에 모두 보내보세요. 새 코드 리뷰 워크플로우를 사용하여 시간 을 절약하고 pull request 작성자 와 불필요하게 주고받는 시간을 줄일 수 있습니다.

pull request에 대한 설명 에서 자세히 확인해보세요.


| Rendering of Jupyter notebooks / Jupyter notebook 렌더링

[ Server ] [ Data Center ]

Bitbucket Server 및 Data Center는 이제 Jupyter notebook 렌더링을 지원 합니다. 개발자와 데이터 과학자 모두 코드, 데이터 및 시각화를 결합한 아이디어를 공유함으로써 각 팀과 협업할 수 있어야 한다는 점을 인식하고 있습니다. 렌더링된 Jupyter notebook을 보려면 Bitbucket의 새 저장소 또는 기존 리포지토리에 파일을 커밋하기만 하면 됩니다.


| Automatically decline inactive pull requests / 비활성 pull request들을 자동으로 거부

[ Data Center ]

목록에서 오픈되어 있는 pull request의 수가 증가함에 따라 원하지 않는 문제들이 발생하곤 합니다. 새로운 자동 제거 요청 옵션을 사용하면 비활성 pull request를 자동으로 제거하여, Bitbucket Data Center의 인스턴스의 성능을 최적화하고 생산성을 높일 수 있습니다. 이 옵션은 기본적으로 설정되며 개별 리포지토리 또는 프로젝트 의 모든 리포지토리에 설정됩니다.

이 기능을 사용하는 방법에 대한 자세한 내용을 보려면 pull requests를 자동으로 거부 를 확인해보세요


| Comment reactions are here 🎉 / 댓글 리액션이 여기 있습니다. 🎉

[ Server ] [ Data Center ]

Comment 에 대한 반응이 좀 더 쉬워졌습니다! 이제 코멘트 리액션을 사용하여 모든 코멘트에 이모티콘을 추가할 수 있습니다. 그래서  👍이모티콘이나 어떤 것을 ✅로 표시하든, 아니면 조금 더 🔥 를 원하든 , 오른쪽 이모지 버튼을 몇 번만 클릭하면 됩니다.

추가 리액션 버튼 을 선택하여 사용해보세요. 자세한 내용은 pull request 에 대해 주석 달기 를 확인해 보세요.


| Diff view  색상 변경 가능

[ Server ] [ Data Center ]

몇가지 중요한 접근성 향상을 위해 노력했습니다. 이 릴리스에는 추가된 라인과 제거된 라인을 파란색과 노란색으 변경할 수 있는 새로운 diff 설정을 포함하였습니다. 해당 기능을 활성화한 후에 즉시 diff 보기에서 색상이 변경된 것을 확인할 수 있습니다.

| Updates to supported platforms /지원되는 플랫폼 업데이트

Bitbucket Server 7.7에서 부터 다음 버전 Git이  추가로 지원됩니다.

  • 서버용 Git 2.29

| Deprecations / 지원 종료

Bitbucket Server 8.0 에서부터 Maria DB 10.1. 의 지원이 종료될 예정입니다.
