Confluence 7.10 릴리즈 노트

| Jira Service Desk is now Jira Service Management / 지라 서비스 데스크 이름 변경

[ Server ] [ Data Center ]

Jira Service Management는 Dev 및 Ops 팀이 빠른 속도로 협업할 수 있도록 지원하므로, 비즈니스 변화에 신속하게 대응하고 우수한 고객 및 직원 서비스 환경을 신속하게 제공할 수 있습니다.

Confluence에서 새로운 이름을 사용하도록 업데이트되었지만 여전히 지식 기반 통합을 포함한 여러기능을 동일하게 제공합니다.

| Clean up your Confluence site / Confluence 사이트 정리

[ Server ] [ Data Center ]

크고 오래된 Confluence 사이트의 눈에 띄는 부작용은 관리하기 어려운 엄청난 양의 컨텐츠가 만들어졌다는 것입니다. 시간이 흐르면, 이것은 정말로 많이 쌓일 것이고, 필요없는 것들을 삭제하는것이 필수적이 될 수 있습니다.

새로운 가이드는 당신에게 이러한 것들을 쉽게 발견할 수 있도록 지원하며 Confluence의 footprint를 줄이기 위한 많은 힌트와 팁을 제공합니다. 보관 공간에서부터 크기가 큰 첨부 파일을 식별하는 것까지, Confluence를 복잡하고 어수선한것으로 부터 자유롭게 해주는 다양한 정보를 제공합니다.

Bitbucket 7.9 릴리즈노트

| Upgrade without downtime / 다운타임 없이 업그레이드

[ Data Center ]

Bitbucket 인스턴스를 최신 상태로 유지하는 것은 필수입니다. 하지만 업그레이드에 필요한 다운타임때문에 1년에 한 번정도만 업그레이드할 수 있고, 이때문에 중요한 버그 및 보안 수정에 대한 업그레이드가 제한되었었습니다.

새로운 Rolling 업그레이드

Bitbucket 데이터 센터 클러스터에서는이제 다운타임 없이 호환되는 버전으로 Rolling 업그레이드를 수행할 수 있습니다.

작업 방법은 다음과 같습니다. 먼저 Bitbucket을 업그레이드 모드(1)로 설정한 다음 각 노드를 차례로 업그레이드하십시오. 각 노드를 업그레이드하는 방법은 배포 환경에 따라 달라집니다.


클러스터 개요 (2) 업그레이드 중에 클러스터에서 발생하는 상황을 정확하게 확인하십시오. 모든 노드에서 새 버전을 실행하면 업그레이드(3)를 완료할 수 있다.

한 번에 한 노드만 아래로 가져가면 된다. 이는 사용자들이 Bitbucket을 업그레이드중에도 코드 검토를 계속할 수 있다는 것을 의미합니다.

업그레이드 자동화

프로세스의 일부를 자동화하고 싶으세요? REST API를 사용하여 업그레이드 모드를 설정 하고 노드 또는 클러스터의 상태를 확인한 후 업그레이드를 완료하세요.

호환 버전

현재는 버그 수정 릴리스에만 Rolling 업그레이드를 사용할 수 있습니다. 예를 들어 Bitbucket 7.9.0에서 7.9.2로  업그레이드를 수행할 수 있지만 비트버킷 7.9.0에서 7.10.0으로 Rolling 업그레이드를 수행할 수는 없습니다. Rolling 업그레이드는 2년 동안 정기적으로 중요한 보안 및 버그 수정을 받는 향후 장기 지원 릴리스 를 사용하실 경우 특히 유용할 것입니다.

Bitbucket을 자신의 On-prem에서 실행하든, AWS 또는 Azure에서 실행하든 다운타임 없이 업그레이드할 수 있습니다.

Rolling 업그레이드에 대해 자세히 알아보기


| Be better prepared for upgrades / 업그레이드를 위한 더 나은 준비과정 가이드

[ Server ] [ Data Center ]

업그레이드에는 많은 계획과 준비가 수반되므로 업그레이드 작업을 쉽게 하기 위해 몇 가지 추가 기능을 추가했습니다.

새로운 업그레이드 전 계획 툴은 필요한 모든 지침을 한 곳에 모아둡니다. Bitbucket 인스턴스를 분석하고, 몇 가지 검사를 실행하고, 해당 인스턴스와 업그레이드 경로에 맞는 업그레이드 가이드를 만들어줍니다.

를 클릭하신뒤> Administration > Plan your upgrade 를 통해 이를 확인해보세요.
