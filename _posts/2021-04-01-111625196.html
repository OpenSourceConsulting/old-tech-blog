아틀라시안의 발표 이후, 오픈소스컨설팅이 구매 고객 대상으로 발송한 아틀라시안 신규 라이선스 제공 중단에 대한 안내를 구독자분들께도 공개합니다. 서버 라이선스의 중단에 대비하여, 3가지 선택 사항에 대한 설명과 가이드를 제시하고 있으니, 자세한 내용은 아래 버튼을 눌러 확인해주세요!
서버 라이센스 중단 안내


Jira Software 8.16.x 릴리즈 노트

| Get more insights into the usage of custom fields - 사용자 정의 필드 사용에 대한 더 많은 정보 보기
[ Data Center ]

사용자 정의 필드가 너무 많아지면 Jira 인스턴스가 복잡해지고 성능에 영향을 미치게 됩니다. 하지만 삭제할 수 있는 필드를 구분하는 것은 쉬운 일이 아닙니다. 이제 모든 사용자 정의 필드에는 중요한 필드인지 그렇지 않은 필요가 없어진 필드인지 구분할 수 있도록 관련된 데이터가 포함됩니다.


두 개의 새로운 열, 이슈들 그리고 마지막 업데이트 일자 는 사용자 정의 필드를 사용하는 문제 수와 마지막으로 값을 업데이트한 시기를 표시합니다. 열 들에 대한 정렬이 가능하기 때문에 삭제하기 위한 필드들을 더욱 빨리 확인할 수 있습니다.. 더 배우기

| Bulk delete custom fields - 사용자 정의 필드 다중 삭제 기능 추가
[ Data Center ]

한 번에 여러 사용자 정의 필드를 선택하고 삭제하는 방법도 추가되었습니다. 특별히 자주 사용하시라는 뜻은 아니지만, 사용자 정의 필드의 새로운 기능을 사용하면 즉시 삭제할 수 있는 필드들을 많이 확인할 수 있을 것으로 예상됩니다.

🌱봄맞이 대청소를 하는 것은 항상 좋습니다. 더 알아보기

| Configure default values for Description field - Description 필드에 대한 기본 값 구성
[ Data Center ]

Description 필드에 대한 기본 값을 설정할 수 있는 기능을 도입했습니다. 지금부터 관리자 는 여러 이슈 유형 및 프로젝트에 기본 값이 있는 컨텍스트를 추가하여 적용할 수 있습니다.

기본 Jira 필드를 그룹 화 할 수 있도록 관리자 메뉴의 이슈 탭의 필드 관련 메뉴에 시스템 필드를 추가했습니다. 현재는 Description 필드에 대해서만 기본 값을 사용하여 다른 컨텍스트를 추가하고 적용할 수 있습니다.


다른 필드의 경우 해당 필드와 연결된 화면을 쉽게 확인하고, 필요한 경우에는 몇 가지 더 추가할 수 있습니다 . 자세히 알아보기

⚠️ Jira Service Management에서는 에이전트가 요청을 생성하거나 편집할 때 Description(설명) 필드의 기본 값을 사용할 수 있지만 고객 포털에는 표시되지 않습니다.

| Manage your DVCS accounts better - DVCS 계정 관리 개선🛠️

이전 Jira 버전에서 DVCS 계정들에 관한 내용을 이미 변경했고, 지금은 좀 더 개선중에 있습니다. 계정 및 리포지토리를 관리하는 성능과 경험을 개선하기 위해 전체 DVCS 계정 페이지를 다시 디자인하고 통합에 사용되는 화면을 정리하고 검색을 추가했습니다.

데이터 센터의 경우 중요한 작업에 대한 감사 로그 이벤트가 추가되었기 때문에 이제 리포지토리를 보다 쉽게 확인하고 문제를 해결할 수 있습니다. 더 배우기

| Let users log in with multiple identity providers - 사용자가 여러 Identity 공급자를 이용하여 로그인할 수 있도록 허용

[ Data Center ]

이제 SAML 및 OpenID Connection 구성을 기반으로 여러 Identity 공급자(IdP)를 이용하여 로그인을 구성할 수 있음을 알려드립니다. 또한 사용자가 로그인할 때 적합한 항목을 선택할 수 있습니다. 그리고 모든 새로운 로그인 방법은 Jira Service Management의 고객 포털에 대해 독립적으로 활성화 시킬 수도 있습니다. 자세히 알아보기

| Check the status of your data pipeline exports - 데이터 파이프라인 내보내기 상태확인

[ Data Center ]


데이터 파이프라인을 통해 Jira에서 현재 상태의 데이터를 내보낸 후 주로 사용하는 비즈니스 인텔리전스 플랫폼(예: Tableau 또는 Power BI)에 제공할 수 있습니다. 자세히 알아보기

이제 지라에서 최근에 내보낸 데이터의 상태를 확인할 수 있습니다. > 시스템 > 데이터 파이프라인 으로 이동하십시오.

| Keep Jira safe by disabling basic authentication - 기본 인증을 사용하지 않도록 설정하여 Jira 안전 유지

복수의 ID 제공자를 추가하는 작업을 진행하면서 로그인 페이지와 API 호출에 기본 인증을 비활성화 하는 방법도 포함 시켰습니다. 기본 인증은 Single Sign-On에 비해 보안성이 낮으므로 SSO로 전환하고 기본 인증을 완전히 사용하지 않도록 설정하는 것이 좋습니다. 자세히 알아보기

| Indexing improvements - 인덱싱 개선

우리는 인덱싱의 개선과 속도를 모두 향상시킬 수 있는 몇 가지 사항을 변경했습니다.

성능 향상, 동일한 높은 일관성

Jira 8.10에서는 인덱싱 성능이 저하되었던 것이 인덱스 일관성이 향상되면서 비용 대비 성능이 개선되었습니다. 이번에는 인덱스 검색기와 캐쉬를 추가로 변경하여 성능을 다시 향상 시켰습니다. 백그라운드 재색인, SLA 재계산, DVCS 재색인 등의 재색인 작업은 10% 이상 빨라졌으며 전체 인덱스 일관성이 유지됩니다. 자세히 알아보기

사용자 인덱싱 요청의 우선 순위 대기열

또한 다양한 유형의 인덱싱 요청을 처리하는 방법도 개선되어 더 이상 사용자 작업에 영향을 미치지 않습니다. 이전 버전에서는 백그라운드 재색인 작업이 단일 대기열을 사용하여 한 번에 많은 인덱싱 작업을 요청할 때 오버플로가 일어날 수 있었습니다. 이제는 이 대기열을 사용자가 생성한 인덱싱 요청을 처리하기 위한 대기열과 다른 모든 요청을 처리하는 대기열로 구분 하였으며, 사용자 요청에 따라서 우선 순위가 부여되며 모든 것이 빠르고 정확하게 정리됩니다 . 자세히 알아보기





Bitbucket Data Center and Server 7.11.x 릴리즈노트

| See your Jira issues on the dashboard - Bitbucket 대시보드에서 Jira Issue 확인
[ Data Center ]

Bitbucket 대시보드가 크게 개선되었습니다. 추가적인 새로운 기능인 Jira Issue 섹션이 있습니다.
Your Issue 항목에서 다음과 같은 내용을 수행 할 수 있습니다. :

  • 맨 위에 있는 최신 업데이트 문제부터 시작하여 자신에게 할당된 모든 열린 Jira Issue를 확인합니다.
  • Issue 키를 쉽게 선택하여 문제의 세부 정보를 엽니다.
  • 작업을 시작할 준비가 되면 수행(Action) 메뉴에서 직접 branch를 작성합니다.

대시보드가 새로운 로직과 함께 변경되어 이제 더 빨리 로드됩니다. 응용프로그램 헤더 상부의 사용자 작업(Your work) 버튼을 사용하여 접근하실 수 있습니다. 새 대시보드에 대한 자세한 내용은 Jira 통합 을 확인해보세요.
