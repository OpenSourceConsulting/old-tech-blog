서버 라이센스 중단 안내


Confluence 7.12.x 릴리즈 노트

| Unlock data and insights about your site - 사이트에 대한 데이터 및 통찰력 확보

[ Data Center ]

이제 Confluence에서 현재 상태 데이터를 내보낸 다음 즐겨찾는 비즈니스 인텔리전스 플랫폼(예: Tableau 또는 PowerBI)) 에 전달해 확인해 보세요. 이 데이터를 사용하여 다음을 수행할 수 있습니다.

  • 사용자 활동에 대한 보고서 및 시각화 생성
  • 조직 리더들에게 팀이 어떻게 Confluence를 사용하고 있는지에 대한 더 나은 이해를 제공
  • Confluence를 최적화하거나 정화하는 것에 대해 결정을 내리실 수 있습니다.

원시 데이터에는 제목, URL, 사용자, 페이지, 첨부 파일 및 주석에 대한 작성 및 수정 날짜 등이 포함됩니다.

REST API를 통해 컨플루언스에서 현재 상태 데이터의 Export 를 트리거할 수 있으며, < 일반 구성 > 데이터 파이프라인 에서 Export 상태를 확인 할 수 있습니다.

초기 Export에는 시간이 걸릴 수 있으며, 첫 Export을 시작하기 전에 알아야 할 성능과 보안에 대한 고려 사항도 확인 할 수 있습니다.

데이터 파이프라인에 대해 자세히 알아보기

| Diagnostic information now included in thread dumps - 스레드 덤프에 진단 정보 포함

[ Data Center ]

Confluence에 문제가 있는 경우, Atlassian Support는 문제를 진단하는 데 도움이 되는 일련의 스레드 덤프를 요청할 수 있었습니다.

이제 스레드 이름에 사용자 이름과 요청 URL이 포함되어 문제를 더 빨리 발견하고 해결할 수 있습니다.

기본적으로 해당기능이 사용되도록 되어 있으며 이러한 세부사항의 내용을 스레드 덤프에 포함시키지 않으려면 > 일반 구성 > 문제 해결 및 지원 도구 > 스레드 진단 에서 진단 기능을 끄시면 됩니다.

Crowd 4.3 릴리즈노트

| Scheduling with cron expressions - Cron 식을 사용한 예약

Crows 4.3부터는 cron 식을 사용하여 디렉토리 동기화를 예약할 수 있습니다. 크론 식을 사용하면 폴링 간격을 사용하는 것보다 디렉토리 동기화를 예약 할 때 더 많은 제어 권한을 얻을 수 있습니다 . 이제 동기화가 시작되는 시간을 매우 구체적으로 정하여 동작하게 할 수 있습니다. 예 를 들어, 매달 둘째 주 금요일 오전 8시 15분 에 동기화를 실행하는 크론 식을 통한 방식으로 작동 가능합니다 . 이 기능 은 Crowd 데이터 센터 에만 사용하실 수 있습니다. Crowd에서 크론 식을 만드는 방법에 대한 자세한 내용은 설명서를 확인해 보세요.



| Pruning delegated directories - 위임된 디렉터리의 사용자를 정리

위임된 디렉토리에서 비활성 사용자 를 정리하여 사용자 데이터베이스를 최적화 하세요 . 위임된 디렉토리에 대해 정리 작업을 구성하시면 , Crowd는 주기적으로 해당 디렉토리에서 비활성 되거나 원격 디렉토리에서 제거된 사용자가 있는지 확인합니다. 위임된 디렉토리 정리 관련 문서를 확인해보세요.

위임된 디렉토리 삭제 기능은 Crowd 4.3버전부터 생성된 기본 기능입니다 . 4.3보다 낮은 Crowd 버전을 사용하시는 경우 다음에서 위임된 디렉터리 제거 플러그인을 설치하여 사용하실 수 있으며, 아틀라시안 마켓플레이스 이 기능을 설치하여 이용하실 수 있습니다.



| Active Object support - 활성 개체 지원

Crowd 4.3은 Active Object에 대한 지원을 제공합니다. Active Object는 Atlassian 제품의 ORM(객체 관계 매핑) 계층으로, Atlassian 애플리케이션에 플러그인으로 구현됩니다. 기존 Bandana 및 PluginSettings API보다 쉽고, 빠르고, 확장성이 뛰어난 데이터 액세스와 스토리지 사용을 가능하게 합니다. 자세한 내용은 Atlassian 개발자 설명서 를 확인해보세요.


| New supported platforms - 지원되는 새로운 플랫폼

다음과 같은 새로운 플랫폼에 대한 지원을 도입했습니다.

  • PostgreSQL 12

  • Microsoft SQL Server 2019

Crows 4.3에서 지원되는 전체 환경 목록은 지원되는 플랫폼 을 확인해보세요.





Bitbucket Data Center and Server 7.12.x 릴리즈노트

| Configure multiple IdPs - 여러 IdP 구성

[ Data Center ]

지금부터 SAML 및 Open에서 다중 ID 제공자(IdP)를 구성할 수 있음을 알려드리게 되어 기쁘게 생각합니다. 이 기능은  Atlassian Data Center 제품에서 사용할 수 있습니다.

  • Jira 8.16 이상 및 Jira Service Management 4.16 이상
  • Bitbucket 7.12 이상

자세한 내용은 다중 ID 공급자 사용 설명서 를 확인해보세요.

| Disable basic authentication - 기본 인증 사용 하지 않기

[ Data Center ] [ Server ]

복수의 ID 제공자를 추가하는 작업을 진행하면서 로그인 페이지와 API 호출에 기본 인증을 비활성화 하는 방법도 포함 시켰습니다. 기본 인증은 Single Sign-On에 비해 보안성이 낮으므로 SSO으로 전환하고 기본 인증을 완전히 사용하지 않도록 설정하는 것이 좋습니다. 기본 인증을 사용하지 않도록 설정하는 방법 에 대해 확인해보세요.


| What else is new? - 또 새롭게 업데이트 된 내용은 무엇일까요?

  • pull request diff에서 이제 컨텍스트 버튼을 위쪽 또는 아래쪽 방향으로 확장할 수 있습니다. 적재 시간을 줄이기 위해 한 번에 25개 라인으로 제한하여 확장되는 모든 라인에 대한 통제도 할 수 있습니다.
  • 끌어오기 요청에 대한 설명을 추가할 때 편집기에 추가한 새로운 형식 지정 도구 모음을 활용하여보세요. 굵게 , 기울임꼴 로, inline code, 취소선 등을 단추 또는 바로 가기 키를 클릭하여 제거하실 수 있습니다.
